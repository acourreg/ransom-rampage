### Key Points
- Research suggests that fintech infrastructure varies by type, with neobanks emphasizing scalable microservices for user-facing services, P2P lenders focusing on risk assessment databases, HFT firms prioritizing low-latency servers, and payment processors relying on API gateways for transaction routing.
- Revenue models show moderate profitability across types, but with risks from regulatory changes and market volatility; neobanks and processors often derive 30-50% from fees and subscriptions, while HFT and P2P rely more on volume-based spreads.
- Vendor dependencies, especially on cloud providers like AWS/GCP and payment networks like Visa, create concentration risks but enable scalability; evidence leans toward hybrid models for resilience.

#### Neobanks
Neobanks like Revolut use cloud-native, API-first architectures for rapid product launches, but face debates over tech vs. banking classification, suggesting balanced views on their hybrid nature.

#### P2P Lenders
Platforms like LendingClub show strong credit performance but highlight risks in peer matching; evidence supports marketplace models for efficiency, though regulatory evolution adds complexity.

#### HFT Firms
Firms like Virtu emphasize low-latency tech amid declining HFT prominence; research indicates medium-frequency shifts, acknowledging infrastructure costs while noting durable liquidity provision.

#### Payment Processors
Processors like Stripe and Wise demonstrate robust growth through diversified streams, but with hedging on vendor lock-in risks; data leans toward API-driven models for global scale.

### Neobanks: Digital-First Banking Evolution
Neobanks represent a shift from traditional banking, offering mobile-centric services without physical branches. Using examples like Revolut, they leverage microservices for agility but must balance innovation with compliance. Typical annual revenue for mid-sized neobanks ranges from $1-3 billion, driven by user growth.

#### Company Archetype Section
- **Fintech Type**: Neobank
- **Typical Infrastructure Nodes** (6-8 components):
  - **API Gateway (Type: API Gateway)**: Handles incoming requests; high throughput, medium defense (via rate limiting), high visibility (detailed logging), moderate cost, medium compliance (supports GDPR via configs). Critical path in revenue as it routes all user interactions.
  - **Auth Server (Type: Auth)**: Manages user authentication; medium throughput, high defense (multi-factor), high visibility (audit trails), cheap cost, high compliance (DORA-ready with token standards). Supporting role, ensures secure access but not direct revenue.
  - **Core Database (Type: Database, e.g., PostgreSQL)**: Stores user data; high throughput, high defense (encryption), high visibility (query monitoring), moderate cost, high compliance (GDPR out-of-box). Critical path for transaction processing.
  - **Cloud Provider (Type: Vendor, e.g., GCP/AWS)**: Infrastructure hosting; high throughput, medium defense (shared), high visibility (cloud monitoring), expensive cost, medium compliance (certified but configurable). Supporting, enables scalability.
  - **Event Bus (Type: Middleware)**: Asynchronous communication; high throughput, medium defense, high visibility (tracing), moderate cost, medium compliance. Critical path for real-time updates.
  - **Payment Network (Type: Vendor, e.g., Visa/Mastercard)**: External processing; high throughput, high defense, medium visibility, expensive cost, high compliance. Critical path for card revenue.
  - **Frontend Server (Type: Server)**: App delivery; medium throughput, medium defense, high visibility, cheap cost, low compliance. Supporting for user engagement.
- **Typical Edge Patterns**: Bidirectional flows from API gateway to auth, then to middleware/event bus for async processing; database reads/writes are synchronous from servers; outbound to vendors (cloud/payment networks) for external dependencies.
- **Critical Nodes**: Core database—if offline, disrupts all revenue streams like payments and subscriptions.

#### Revenue Flow Section
- **Typical Revenue Streams** (3-4 flows per type):
  - **Card Payments**: Node chain: API Gateway → Auth Server → Frontend Server → Payment Network → Core Database. Typical revenue $500M-$1B annually (e.g., Revolut's $887M in 2024). Shared: API Gateway, Core Database. Bottleneck: Payment Network (high but external latency).
  - **FX Transfers**: Node chain: API Gateway → Event Bus → Cloud Provider → Core Database. Typical revenue $300M-$600M annually (e.g., Revolut's $540M). Shared: Core Database. Bottleneck: Event Bus (medium throughput for real-time).
  - **Subscriptions**: Node chain: API Gateway → Auth Server → Core Database. Typical revenue $300M-$500M annually (e.g., Revolut's $541M). Shared: API Gateway, Core Database. Bottleneck: Auth Server (medium for verification).
  - **Wealth Services**: Node chain: API Gateway → Middleware → Core Database → Cloud Provider. Typical revenue $100M-$600M annually (e.g., Revolut's $647M). Shared: Core Database. Bottleneck: Middleware (medium for integrations).

### P2P Lenders: Marketplace Lending Platforms
P2P lenders like LendingClub connect borrowers and investors digitally, with evidence showing superior credit scoring over traditional models, though dependent on regulatory compliance for growth. Typical annual revenue $500M-$1B for established platforms.

#### Company Archetype Section
- **Fintech Type**: P2P Lender
- **Typical Infrastructure Nodes** (6-8 components):
  - **API Gateway (Type: API Gateway)**: Loan matching requests; medium throughput, medium defense, high visibility, moderate cost, medium compliance. Critical path for platform matching.
  - **Auth Server (Type: Auth)**: Borrower/investor verification; low throughput, high defense, high visibility, cheap cost, high compliance (GDPR-ready). Supporting for trust.
  - **Risk Database (Type: Database)**: Credit scoring data; medium throughput, high defense, high visibility, moderate cost, high compliance. Critical path for loan approval revenue.
  - **Cloud Provider (Type: Vendor, e.g., AWS)**: Scalable hosting; medium throughput, medium defense, high visibility, moderate cost, medium compliance. Supporting for operations.
  - **Payment Network (Type: Vendor, e.g., ACH)**: Fund transfers; medium throughput, high defense, medium visibility, expensive cost, high compliance. Critical path for disbursements.
  - **Middleware (Type: Middleware)**: Integration layer; medium throughput, medium defense, high visibility, moderate cost, medium compliance. Supporting for data flow.
  - **Analytics Server (Type: Server)**: Portfolio management; low throughput, medium defense, high visibility, cheap cost, low compliance. Supporting for investor returns.
- **Typical Edge Patterns**: Unidirectional from API gateway to auth/risk database for verification; bidirectional to middleware for matching; outbound to vendors for funds.
- **Critical Nodes**: Risk Database—if offline, halts loan origination, disrupting core revenue.

#### Revenue Flow Section
- **Typical Revenue Streams** (3-4 flows per type):
  - **Origination Fees**: Node chain: API Gateway → Auth Server → Risk Database → Payment Network. Typical revenue $200M-$500M annually (e.g., LendingClub's model). Shared: Risk Database. Bottleneck: Risk Database (medium for scoring).
  - **Servicing Fees**: Node chain: Middleware → Analytics Server → Core Database. Typical revenue $100M-$300M annually. Shared: Core Database. Bottleneck: Analytics Server (low throughput).
  - **Investor Fees**: Node chain: API Gateway → Payment Network → Cloud Provider. Typical revenue $50M-$150M annually. Shared: API Gateway. Bottleneck: Payment Network.
  - **Interest Spreads**: Node chain: Risk Database → Middleware → Payment Network. Typical revenue $100M-$400M annually. Shared: Risk Database. Bottleneck: Middleware.

### HFT Firms: High-Speed Trading Operations
HFT firms like Virtu focus on low-latency infrastructure amid market debates on volatility impacts; evidence supports liquidity benefits but notes high entry barriers. Typical annual revenue $2-4B for leaders.

#### Company Archetype Section
- **Fintech Type**: HFT Firm
- **Typical Infrastructure Nodes** (6-8 components):
  - **Trading Server (Type: Server)**: Execution engine; high throughput, high defense, high visibility, expensive cost, medium compliance. Critical path for all trades.
  - **Data Provider (Type: Vendor, e.g., Bloomberg)**: Market feeds; high throughput, medium defense, high visibility, expensive cost, low compliance. Supporting for signals.
  - **Colocation (Type: Infrastructure)**: Low-latency hosting; high throughput, high defense, medium visibility, expensive cost, medium compliance. Critical path for speed.
  - **API Gateway (Type: API Gateway)**: Exchange connections; high throughput, medium defense, high visibility, moderate cost, medium compliance. Critical path.
  - **Risk Database (Type: Database)**: Position tracking; high throughput, high defense, high visibility, moderate cost, high compliance. Supporting but vital.
  - **Middleware (Type: Middleware)**: Event processing; high throughput, medium defense, high visibility, moderate cost, medium compliance. Critical for real-time.
  - **Cloud Provider (Type: Vendor, e.g., AWS)**: Backup analytics; medium throughput, medium defense, high visibility, moderate cost, medium compliance. Supporting.
- **Typical Edge Patterns**: Unidirectional high-speed feeds from data providers to servers; bidirectional via middleware to exchanges; low-latency loops for risk checks.
- **Critical Nodes**: Trading Server—if offline, stops all revenue-generating trades.

#### Revenue Flow Section
- **Typical Revenue Streams** (3-4 flows per type):
  - **Market Making**: Node chain: Data Provider → Middleware → Trading Server → API Gateway. Typical revenue $1-2B annually (e.g., Virtu's model). Shared: Trading Server. Bottleneck: Trading Server (high but latency-sensitive).
  - **Execution Services**: Node chain: API Gateway → Risk Database → Cloud Provider. Typical revenue $500M-$1B annually. Shared: API Gateway. Bottleneck: Risk Database.
  - **Technology Licensing**: Node chain: Middleware → Cloud Provider. Typical revenue $100M-$300M annually. Shared: Middleware. Bottleneck: Cloud Provider.
  - **Commissions**: Node chain: API Gateway → Trading Server. Typical revenue $200M-$500M annually. Shared: API Gateway. Bottleneck: API Gateway.

### Payment Processors: Transaction Facilitation Giants
Processors like Stripe and Wise show strong diversification, with hedging on global expansion risks; evidence supports API-first models for efficiency amid vendor debates. Typical annual revenue $3-5B for top players.

#### Company Archetype Section
- **Fintech Type**: Payment Processor
- **Typical Infrastructure Nodes** (6-8 components):
  - **API Gateway (Type: API Gateway)**: Transaction routing; high throughput, high defense, high visibility, moderate cost, high compliance. Critical path for all flows.
  - **Auth Server (Type: Auth)**: Fraud checks; high throughput, high defense, high visibility, cheap cost, high compliance. Supporting for security.
  - **Transaction Database (Type: Database)**: Ledger storage; high throughput, high defense, high visibility, moderate cost, high compliance. Critical path.
  - **Cloud Provider (Type: Vendor, e.g., AWS)**: Scalable hosting; high throughput, medium defense, high visibility, expensive cost, medium compliance. Supporting.
  - **Payment Network (Type: Vendor, e.g., Visa)**: Clearing; high throughput, high defense, medium visibility, expensive cost, high compliance. Critical path.
  - **Middleware (Type: Middleware)**: Orchestration; high throughput, medium defense, high visibility, moderate cost, medium compliance. Supporting.
  - **Analytics Server (Type: Server)**: Reporting; medium throughput, medium defense, high visibility, cheap cost, low compliance. Supporting.
- **Typical Edge Patterns**: Inbound from API gateway to auth/middleware; synchronous to database; outbound to vendors for settlement.
- **Critical Nodes**: API Gateway—if offline, blocks all transaction revenue.

#### Revenue Flow Section
- **Typical Revenue Streams** (3-4 flows per type):
  - **Transaction Fees**: Node chain: API Gateway → Auth Server → Payment Network → Transaction Database. Typical revenue $2-3B annually (e.g., Stripe's model). Shared: API Gateway, Database. Bottleneck: Payment Network.
  - **Subscriptions**: Node chain: API Gateway → Middleware → Analytics Server. Typical revenue $500M-$1B annually. Shared: API Gateway. Bottleneck: Middleware.
  - **FX Conversions**: Node chain: API Gateway → Cloud Provider → Transaction Database. Typical revenue $300M-$600M annually (e.g., Wise). Shared: Database. Bottleneck: Cloud Provider.
  - **Value-Added Services**: Node chain: Middleware → Analytics Server → Cloud Provider. Typical revenue $200M-$500M annually. Shared: Middleware. Bottleneck: Analytics Server.
