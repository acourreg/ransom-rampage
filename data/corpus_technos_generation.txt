#### Vault-Secret-Manager

**id**: secrets_manager
**category**: middleware
**subcategory**: security

**description**: The high-security locker for your keys, passwords, and certificates. It’s like a digital Swiss bank. If you hardcode your DB password in Git, this service is here to tell you how much you failed.

**game_stats**:
- throughput: 6-8
- defense: 10-10
- visibility: 6-8
- cost: 3-6
- compliance_score: 10-10

**strengths**:
- Automatic secret rotation (passwords change every 30 days).
- Dynamic credentials: your app gets a temporary password that dies in 1 hour.
- Encryption-as-a-Service: keeps raw PII off your disks.

**weaknesses**:
- "The Keys to the Kingdom": if this is breached, you're 100% dead.
- Bootstrapping problem: "Where do I store the key to the secret manager?"
- Latency: fetching a secret for every DB connection is slow.

**typical_edges**:
- → relational_db: provides the credentials for app access.
- → payment_engine: protects the API keys for the card networks.
- ↔ auth_service: secures the JWT signing keys.

**sector_affinity**: neobank, hft, payments
**flow_roles**: security

**cost_range_eur_k**: 18-32 €K/turn

**category**: middleware
**subcategory**: analytics

**description**: The black box recorder. It collects every log, trace, and metric. It’s useless until something goes wrong; then it’s the only way to find the needle in the haystack of your 50 microservices.

**game_stats**:
- throughput: 7-10
- defense: 4-6
- visibility: 10-10
- cost: 5-10
- compliance_score: 7-9

**strengths**:
- Real-time alerting: "Why is the 500-error rate at 20%?"

#### Auto-Deploy-Pipeline-Orchestrator

**id**: cicd_pipeline
**category**: middleware
**subcategory**: integration

**description**: The conveyor belt that takes code from a developer's laptop and throws it into production. It runs tests, checks for security holes, and automates the destruction of your stable environment.

**game_stats**:
- throughput: 5-8
- defense: 5-7
- visibility: 7-9
- cost: 3-6
- compliance_score: 8-10

**strengths**:
- "Ship Fast": deploy code 50 times a day without human touch.
- Automated security scanning: finds old libraries before hackers do.
- Standardizes the environment: "It works on my machine" is no longer an excuse.

**weaknesses**:
- Supply Chain Attacks: if your CI tool is hacked, they can inject code into your bank.
- "The Red Build": one flaky test can stop all progress for the whole team.
- Complexity: the pipeline code is often more complex than the app.

**typical_edges**:
- → service_mesh: pushes new routing rules during a deploy.
- → secrets_manager: pulls the keys needed for build-time secrets.
- ← monitoring_logging: reports if the new deploy is crashing everything.

**sector_affinity**: neobank, hft, p2p, payments
**flow_roles**: support, security

**cost_range_eur_k**: 20-40 €K/turn
